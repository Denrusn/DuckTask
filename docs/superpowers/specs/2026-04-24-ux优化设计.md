# DuckTask UX 优化设计

## 概述

对 DuckTask 进行 5 项 UX 优化，涉及卡片样式、输入方式、日志确认逻辑、日志中心导航和锁屏强提醒处理。

---

## Issue 1: 卡片灰色边框优化

### 现状
`TaskCard` 使用 `elevation = 2.dp` + 半透明背景 `accent.copy(alpha = 0.06f)`，阴影在浅色背景下显得灰暗。

### 修改
- `elevation` 从 `2.dp` 改为 `1.dp`，降低阴影强度
- 背景从 `accent.copy(alpha = 0.06f)` 改为接近白色的纯色背景，保持与边框的对比
- 边框 `BorderStroke(1.dp, accent.copy(alpha = 0.14f))` 保持不变

### 涉及文件
- `app/src/main/java/com/ducktask/ui/screens/MainScreen.kt` — `TaskCard` composable

---

## Issue 2: 悬浮按钮 + 底部弹出输入

### 现状
`InputCard` 固定显示在首页顶部，占据固定空间。

### 修改
- 首页移除固定的 `InputCard`，改为 `ExtendedFloatingActionButton`（右下角，显示"添加提醒"）
- FAB 点击后底部滑出 `InputCard`（使用 `ModalBottomSheet` 或 `BottomSheetScaffold`）
- 提交成功后自动收起，back 键可收起
- 动画使用 `animateContentSize` + `AnimatedVisibility` 实现滑出/收起效果

### 涉及文件
- `app/src/main/java/com/ducktask/ui/screens/MainScreen.kt` — `HomeContent`、`InputCard` 重构

## Issue 3: 执行记录手动完成更新

### 现状
用户点击"完成"后，任务状态变为 `COMPLETED`，但 `ReminderExecutionLog` 的 `acknowledgedAt` 和 `dismissMethod` 未更新，执行记录页面仍显示"待处理"。

### 修改
`TaskRepository.markAsDone()` 方法中，在更新任务状态后：
1. 查询该 `taskId` 最新一条 `ReminderExecutionLog`（`acknowledgedAt IS NULL`）
2. 调用 `reminderLogDao.acknowledge(logId, System.currentTimeMillis(), "手动完成")`

### 涉及文件
- `app/src/main/java/com/ducktask/data/repository/TaskRepository.kt` — `markAsDone()`
- `app/src/main/java/com/ducktask/data/local/ReminderLogDao.kt` — 新增查询方法

## Issue 4: 日志中心图标替代 FilterChip

### 现状
主页 TopAppBar 有执行记录和运行日志图标按钮，日志中心页面只有 `FilterChip` 切换。

### 修改
- `LogPageContent` 移除 `FilterChip`
- 在 TopAppBar 的 actions 中加入两个 `IconButton`（`History` 和 `Description`），点击切换对应 Tab
- 行为与主页 TopAppBar 一致

### 涉及文件
- `app/src/main/java/com/ducktask/ui/screens/MainScreen.kt` — `LogPageContent`、`MainDestination.LOGS` 的 TopAppBar actions

## Issue 5: 解锁后立即弹窗

### 现状
强提醒触发时若检测到锁屏（`isDeviceLocked = true`），`StrongReminderOverlayService.startIfPossible()` 返回 `false`，只发送通知，不弹窗。锁屏期间通知栏能收到提醒，但解锁后不会自动弹出悬浮窗。

### 修改
新增 `DeviceUnlockReceiver`：
1. 监听 `Intent.ACTION_USER_PRESENT`（用户解锁屏幕）
2. 解锁时查询是否存在 `status = ALERTING` + `reminderMode = STRONG` 的未处理任务
3. 若存在，启动 `StrongReminderOverlayService` 显示悬浮窗
4. 注册方式：在 `DuckTaskApp`（Application）中通过 `registerReceiver` 动态注册，不依赖 AndroidManifest

### 涉及文件
- `app/src/main/java/com/ducktask/DuckTaskApp.kt` — 注册/注销 `DeviceUnlockReceiver`
- 新建 `app/src/main/java/com/ducktask/DeviceUnlockReceiver.kt`
