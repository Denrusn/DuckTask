# DuckTask 智能定时提醒APP - 需求文档（文字版）

---

## 一、产品概述

### 1.1 产品定位
**DuckTask** 是一款基于自然语言处理的智能定时提醒应用。用户可以通过日常语言（如"周五晚上提醒我打电话给老妈"）快速创建提醒任务，系统自动解析时间并执行定时提醒。

### 1.2 产品名称
- **应用名称**：DuckTask
- **应用图标**：鸭子形象（待设计）
- **品牌口号**：「准时提醒，就像鸭子一样准时！」（待定）

### 1.3 核心价值
- **零学习成本**：用自然语言创建提醒，无需选择日期时间控件
- **灵活的重复规则**：支持一次性、每日、每周、每月、每年等多种重复模式
- **可靠的后台执行**：应用退出后仍能准时触发提醒

---

## 二、技术架构概述

根据原始脚本逻辑，**DuckTask** 安卓APP需要实现以下核心模块的对应功能：

| Python模块 | APP对应模块 | 功能说明 |
|-----------|------------|---------|
| scheduler_parser.py | NLP解析引擎 | 自然语言分词、规则匹配、时间提取 |
| scheduler.py | 任务调度引擎 | APScheduler → Android AlarmManager/WorkManager |
| reminder_service.py | 提醒服务层 | 业务逻辑、触发器转换、提醒执行 |
| PostgreSQL | 本地数据库 | SQLite/Room 存储任务持久化 |

### 3.1 应用名称与标识

- 应用包名：`com.ducktask.app`
- 应用标题显示为"DuckTask"

- 设置页面标题："DuckTask 设置"

### 3.2 自然语言解析模块（NPLParser）

**功能描述**：将用户输入的自然语言文本解析为结构化的提醒数据。

**输入示例**：
```
"周五晚上提醒我打电话给老妈"
"两个星期后提醒我去复诊"
"每月20号提醒我还信用卡"
"每天上午9点提醒我喝水"
```

**输出结构**：
```json
{
  "event": "打电话给老妈",
  "reminder_time": "2024-XX-XX 20:00:00",
  "repeat": {"weeks": 1},
  "description": "原始输入文本"
}
```

**支持的解析规则**：

| 类别 | 支持的表达 | 示例 |
|-----|-----------|-----|
| 绝对时间 | 年月日时分秒 | 2024年5月1日下午3点 |
| 相对天数 | 今天/明天/后天/N天后 | 3天后、周五、下周一 |
| 相对时段 | 上午/下午/晚上+时间 | 明天下午3点、明天晚上8点 |
| 重复-年 | 每年/N年 | 每年5月1号、每隔两年 |
| 重复-月 | 每月/N个月 | 每月20号、每隔三个月 |
| 重复-周 | 每周/N周 | 每周一、每隔两周 |
| 重复-天 | 每天/N天 | 每天早上9点、每隔3天 |
| 重复-小时 | 每隔N小时 | 每隔2小时 |
| 中文数字 | 一/二/三...零 | 三天后、三点二十 |

**边界处理**：
- 过去时间自动拒绝并提示用户
- 时间跨度超限（如100年）给出友好提示
- 无法解析时提示用户调整表达方式

---

### 3.3 任务调度引擎模块（SchedulerEngine）

**功能描述**：管理所有提醒任务的添加、删除、查询、持久化和执行。

**核心功能**：

**3.3.1 任务添加**
- 将NLP解析后的结构化数据转换为系统trigger
- 生成唯一任务ID（UUID）
- 注册到调度队列

**3.3.2 Trigger类型转换**

| repeat参数 | Trigger类型 | 说明 |
|-----------|-------------|-----|
| 无repeat（一次性） | DateTrigger | 指定日期时间触发一次 |
| days/weeks/months/years | CronTrigger | 按cron表达式周期执行 |
| hours/minutes/seconds | IntervalTrigger | 按固定间隔重复 |

**3.3.3 任务持久化**
- 每次添加/删除任务，同步写入本地数据库
- 应用重启后自动恢复所有未过期任务
- 数据库字段：
 - task_id（UUID）
 - who（用户标识）
 - event（事件描述）
 - description（原始输入）
 - reminder_time（首次提醒时间）
 - repeat（重复规则JSON）
 - trigger_type（触发器类型）
 - create_time（创建时间）
 - status（状态：待执行/已完成/已删除）

**3.3.4 任务删除**
- 支持按任务序号删除（如"删除提醒1"）
- 支持按task_id精准删除
- 删除时同步更新数据库

**3.3.5 任务查询**
- 获取当前用户所有提醒任务
- 按创建时间降序排列
- 显示下次触发时间

---

### 3.4 提醒执行模块（ReminderExecutor）

**功能描述**：当触发时间到达时，执行提醒通知。

**执行流程**：
1. 调度器检测到任务触发时间到达
2. 执行execute_reminder回调
3. 构建提醒消息内容
4. 发送系统通知（Notification）
5. 如为重复任务，计算并更新下次触发时间

**通知内容格式**：
```
🕒 [事件描述]

备注：[原始输入]
下次提醒时间：3天后 2024-05-01 09:00
重复：每周
```

## 五、关键算法说明

### 5.1 自然语言解析算法

**分词与词性标注**
- 使用中文分词库（Android可使用HanLP或自定义词典）
- 过滤停用词（的、了、在等）
- 识别时间相关词性和数字

**规则匹配优先级**
```
1. 重复规则优先（每/每隔 + 数字 + 单位）
2. 年份规则（今年/明年/后年/数字年）
3. 月份规则（下个月/数字个月）
4. 日期规则（今天/明天/后天/数字天后）
5. 星期规则（周一/周五/下周一）
6. 时间点规则（上午/下午/晚上 + 数字点）
7. 截取剩余文本作为事件描述
```

### 5.2 Trigger转换算法

```
输入：repeat字典 + reminder_time时间

处理逻辑：
1. 如果repeat为空：
   - 判断reminder_time是否在当前时间之后
   - 是 → DateTrigger
   - 否 → 抛出异常（时间已过）

2. 如果repeat包含 years/months/days：
   - 构建CronTrigger
   - years → year参数
   - months → month参数 + day参数
   - days → day参数
   - 设置hour和minute

3. 如果repeat包含 hours/minutes/seconds：
   - 构建IntervalTrigger
   - 将repeat转换为总秒数
   - 设置start_date

输出：trigger对象 + trigger_type字符串
```

### 5.3 人性化时间显示算法

将datetime转换为自然语言：
```
2024-05-15 10:30 → 3天后 10:30
2024-05-15 09:00 → 明天 09:00
2024-05-16 20:00 → 周五晚 20:00
```

## 六、非功能性需求

### 6.1 性能要求
- 自然语言解析响应时间 < 500ms
- 任务添加响应时间 < 200ms
- 通知触发延迟 < 5秒（系统允许范围内）

### 6.2 可靠性要求
- 应用进程被杀死后，定时任务仍能执行
- 使用AlarmManager + BroadcastReceiver机制
- 重复任务执行后自动更新下次触发时间
- 数据库记录与调度器状态保持一致

### 6.3 兼容性要求
- 最低支持Android 8.0（API 26）
- 目标版本Android 14（API 34）
- 申请必要的运行时权限（通知权限、后台弹出界面权限）

### 6.4 用户体验
- 支持深色/浅色主题
- 支持语音输入创建提醒
- 支持小组件快捷查看提醒
- 支持与其他APP（如日历）数据同步（可选）
- 应用内所有提示文案统一使用"DuckTask"作为品牌名称
